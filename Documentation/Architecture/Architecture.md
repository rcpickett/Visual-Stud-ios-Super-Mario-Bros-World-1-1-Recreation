\# Game Architecture



\## Overview



The project was designed using an object-oriented architecture separating gameplay responsibilities into independent systems.



\## Core Systems



\### Game Loop



Responsible for:

\- Updating game state

\- Processing input

\- Updating gameplay objects

\- Rendering frames



\### Player System



Handles:

\- Character behavior

\- Movement states

\- Player interaction



\### Collision System



Responsible for:

\- Object collision detection

\- Environment interaction

\- Gameplay boundaries



\### Sprite System



Manages:

\- Sprite loading

\- Rendering

\- Animation states



\### Game State System



Controls:

\- Menus

\- Gameplay

\- High scores

\- End states



\## Content Pipeline



XML content files were used to manage game data and assets.
